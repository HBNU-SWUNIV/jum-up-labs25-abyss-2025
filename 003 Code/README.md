## Teammate Project Background

- 필요성
  - 여러 도메인의 지표(품질, 설비, 환경, 학습 데이터 등)를 **한 화면에서 통합 시각화**하고 팀별 맞춤 템플릿으로 재사용.
  - 신규 도메인/지표 추가 시 **플러그인 교체만으로 확장**(코어 수정 최소화).
  - 실시간 이벤트(알람/임계치) 기반 **즉시 대응**과 권한·이력관리 필요.

- 기존 해결책의 문제점
  - 범용 BI 도구는 강력하지만 **학습 곡선이 가파르고** 도메인 특화 옵션 모델링이 어려움.
  - 위젯/차트 수준의 조합에 머물러 **데이터 전처리·집계·권한**을 일관되게 다루기 어려움.
  - 실시간과 배치를 섞을 때 **성능·캐시·동시성 제어** 부재로 지연과 비용 증가.

## System Design

- System Requirements
  - 기능
    - **플러그인 타입/인스턴스 관리**: `typeId`, `optionSchema`, `queryTemplate`, `rendererCode`.
    - **대시보드 편집기**: Drag&Drop 레이아웃, 그리드/리사이즈, 공유/템플릿 저장.
    - **데이터 연동**
      - 파일형: 엑셀/CSV 업로드 → 표준 스키마 자동 매핑(컬럼 미스매치 가이드).
      - DB형: MySQL 쿼리, 파라미터 바인딩, 캐시(선택).
      - 실시간형: WebSocket, 임계치 알림.
    - **권한/감사**: 역할(관리자/편집자/뷰어), 변경 이력/대시보드 버전.
    - **운영**: 플러그인 마켓(등록/승인/비활성화), 메트릭 노출.
  - 비기능
    - **성능**: 주요 위젯 초기 로드 ≤ 3s, 재조회 ≤ 2s(캐시/지연 로딩).
    - **확장성**: 신규 플러그인 추가 시 코어 무변경 원칙(인터페이스 준수).
    - **안정성/보안**: 플러그인의 DB 직접접근 금지(제어 레이어), 입력 검증.
    - **테스트/품질**: FE 컴포넌트 테스트, API 통합 테스트, 주요 시나리오.
- 아키텍처 개요
  - **Frontend(React + Vite)**: 레이아웃/플러그인 UI, 옵션→`queryTemplate` 바인딩, 실시간 구독.
  - **Plugin Server(Node.js)**: 타입 레지스트리, 렌더/실행 파이프라인, 서드파티 플러그인 로딩.
  - **Backend API(Spring Boot)**: 인증·권한, 파일 표준화, 쿼리 실행 프록시, 감사 로그.
  - **DB/Cache**: MongoDB & MySQL.
- 데이터 흐름(예시)
  - 파일형: 사용자 업로드 → 스키마 추론/정규화 → 저장(객체 스토리지) → 조회 시 변환/집계 → 시각화.
  - 스트림형: 데이터 소스 → DB 적재/캐시 → 임계치 평가 → WebSocket 푸시 → 실시간 위젯 반영.
- 제공 플러그인(샘플)
  - 차트: Line/Bar/Area/Pie, 테이블(페이징/컬럼 설정).
  - 분석: 이동평균, 누적/전년동기, 상관/피벗 요약(서버에서 계산).
  - 유틸: 텍스트/마크다운, 이미지, 알림 피드, 임계치 하이라이트.

## Case Study

- Description
  - **ABS(플러그인 기반 대시보드)** 를 적용해 두 가지 경로를 한 프로젝트에서 운용:
    1) **파일형 PoC**: 학과/동아리 데이터 → 엑셀 업로드 → 표준 스키마 변환 → KPI/추이/피벗 대시보드 생성  
    2) **실시간형 PoC**: 엣지/서버 로그 → DB 적재 → Redis 캐시 → 임계치 이벤트 시 WebSocket 알림 → 운영 패널 반영
  - **권한 시나리오**: 관리자(템플릿 배포) ↔ 편집자(필터/옵션 저장) ↔ 뷰어(읽기 전용 링크).
  - **운영 시나리오**: 플러그인 신규 등록(검증/승인), 버전 롤백, 문제 플러그인 격리.

## Conclusion

- **유연성**: 타입/인스턴스 분리와 표준 옵션 스키마로 도메인 전환 비용 최소화.
- **재사용성**: 템플릿/프리셋 공유, 데이터-표현 분리로 팀 간 베스트프랙티스 축적.
- **실효성**: 파일형→실시간형으로 **저비용 확장 경로** 제공, 초기 도입 장벽↓.
- **운영성**: 권한/감사/헬스체크/메트릭으로 현업 운영·감사 대응 용이.
