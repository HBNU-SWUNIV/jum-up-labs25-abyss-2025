## Project Background
- 최근 재활병원이나 케어시설 등에서 환자의 낙상, 비정상적인 움직임, 응급 상황을 **실시간으로 감지할 수 있는 시스템**의 필요성이 커지고 있다.
  </br>
  </br>
- 기존 CCTV 기반 모니터링은 단순한 영상 저장이나 사후 확인에 그치기 때문에, 관리자가 즉각적인 대응을 하기 어렵다는 한계가 있었으며 이에 본 프로젝트에서는 **인공지능(AI)** 과 **센서 데이터**, **컴퓨터 비전 기술**을 결합하여 환자의 동작을 분석하고 위험 상황 발생 시 관리자에게 즉시 알림을 제공하는 **실시간 모니터링 및 알림 시스템**을 구축하고자 한다.
  </br>
  </br>
- 카메라로부터 실시간으로 수집되는 영상을 분석하여 낙상이나 비정상 행동을 인식하고, WebRTC 기반 스트리밍 서버를 통해 관리자 대시보드로 경고를 전달한다. 또한 MongoDB 및 GraphQL을 활용하여 데이터를 효율적으로 관리하고, 시각화된 대시보드 화면을 통해 관리자가 상황을 직관적으로 확인할 수 있도록 설계하였다.

## System Model

- 시스템은 **엣지 카메라 모듈**, **백엔드 서버**, **프론트엔드 대시보드**로 구성된다.  

<img width="1980" height="686" alt="image" src="https://github.com/user-attachments/assets/7668dc3c-ae3f-4dd8-96e9-cc0699b9a453" />


- **엣지 카메라 모듈**  
  - YOLO 기반 객체 탐지 모델을 이용하여 사용자의 움직임을 분석  
  - 위험 상황 발생 시 실시간 데이터를 백엔드 서버로 전송  

- **백엔드 서버**  
  - FastAPI 및 Node.js 기반으로 실시간 데이터 스트림 관리  
  - WebSocket과 WebRTC를 통해 데이터 전송 및 모니터링 구현  
  - MongoDB에 환자별 기록 및 이벤트 로그 저장  
  - GraphQL API를 통해 데이터 접근 및 조회 성능 최적화  

- **프론트엔드 대시보드 (React 기반)**  
  - 실시간 영상 및 위험 알림 표시  
  - 환자별 상태 데이터와 시각화된 통계 제공  
  - 관리자 맞춤형 UI로 직관적인 데이터 관리 가능  

## Numerical Results
- 현재 구축된 시제품 수준의 시스템을 테스트한 결과는 다음과 같다.  

- 실시간 영상 스트리밍 지연: **평균 0.8초 이하 유지**  
- 낙상 및 비정상 행동 감지 정확도: **90% 이상**  
- GraphQL 기반 데이터 관리 효율성: **기존 REST API 대비 약 20% 향상**  
- 대시보드 실시간 반영 속도: **3초 이내 반응**  

테스트는 로컬 네트워크 기반 환경에서 진행되었으며,  
실제 병원 환경에서도 적용 가능한 수준의 안정성과 성능을 확인하였다.

## Conclusion
- 본 연구를 통해 개발된 실시간 환자 모니터링 시스템은 기존 CCTV 감시 방식의 한계를 보완하고,  
AI 기반 영상 분석과 실시간 데이터 스트리밍을 결합하여 **즉각적인 상황 대응이 가능한 구조**를 구현하였다.  

이를 통해 낙상, 무움직임, 비정상 행동 등의 위험 상황을 빠르게 인식하고 관리자에게 즉시 알림을 전송할 수 있었다.  
또한 맞춤형 대시보드 설계를 통해 관리자 중심의 효율적인 모니터링이 가능함을 검증하였다.  

향후에는 다양한 환경(병원, 가정, 요양시설 등)에 적용할 수 있도록  
모듈 경량화와 하드웨어 최적화를 진행할 예정이다. 

## Project Outcome
- **기술적 성과** – 실시간 낙상 감지 및 AI 기반 행동 분석 알고리즘 검증
- **시스템 성과** – WebRTC 및 GraphQL 기반 실시간 스트리밍·데이터 통합 구조 완성
- **학술적 성과** – 한국시뮬레이션학회 및 한국전기전자학회 학술대회 발표
